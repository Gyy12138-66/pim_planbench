# Item 级区分度分析与 26 题处置清单（v0.4 修订项 #1 前置）

- 数据：`scores/pilot_v0.3/ModelOutput_all_existing_scored_v0.3.csv`（6 模型 × 26 题 × 5 facet，v0.3 冻结评分器）
- 明细指标：`item_metrics_v0.4.csv`（本目录）
- **统计声明**：n=6 模型，全部指标是分诊工具而非推断统计；修订项 #3（每模型 ×5 采样）
  落地后重算并附 bootstrap CI。本文档的处置决定以指标为主、题目设计判断为辅，逐题给出理由。

## 1. 指标定义

| 指标 | 定义 | 病理判读 |
|---|---|---|
| spread | 6 模型总分最大−最小 | ≤3 = 饱和（无法承载模型间比较） |
| item_rest_rho | 本题得分与"总分−本题"的 Spearman | ≈0 或负 = 本题排序与整体不一致 |
| facet 满分率 | 该题各 facet 得 5 分的比例 | ≈1.0 = 该 facet 在该题上送分 |
| mean_dev | 题均分对全局均分（21.2）的偏离 | 难度层间应有梯度，现无 |

## 2. 全局诊断（框架级问题，先于逐题处置）

1. **难度轴失效**：easy 21.3 / medium 20.7 / hard 21.4 / hard_plus 21.6——难度递增分数不动。
   病因：hard 题的分数并不取决于"是否处理了该题的困难点"。修复原则见 §3。
2. **facet 送分不均**：全局满分率 38.1%（目标 <15%）；Model Choice 57.7%、Training
   Strategy 48.7% vs Problem Formalization 26.9%。**PF 低饱和的原因是 `pf_strict_cap`**
   （要求已知/未知区分、任务类型、缺失规格处理的结构完整性检查，780 行中触发 86 次）——
   这是全评分器唯一的"结构核验"型规则，也是唯一有效抑制饱和的规则。
   **结论：把 PF 式结构严格化推广到 MC/TS facet，并把 cap 从线索缺席检测升级为断言核验**
   （修订项 #2，见 `scorer v0.4` 工作）。
3. **hard-trap cap 名存实亡**：6 道带任务特异 trap 的题（015/017/020/023/025/026），
   trap 仅在 023 触发 3 次、015 触发 2 次、025 触发 1 次；017/020/026 零触发。
   模型只要**提到**关键词（chemical potential、Fourier features、operator learning）
   就能绕过缺席检测。
4. **有效对照**：spread 前二的 023（8 分，ρ=0.70）与 025（7 分）恰是 trap 真正触发过的题。
   共同结构 = 存在一个**二元、后果严重、模板文本绕不过的建模决策**（守恒变量/激波处理；
   磁散度控制），且 cap 钉住它。这是重写的模板，下称**决策支点**。

## 3. 重写原则（喂给修订项 #1/#6 的设计规则）

每道非 easy 题必须具备：
- **P1 决策支点**：一个可判定的任务特异建模决策，做对/做错导致执行成败分化；
  写进任务特异 cap。锚定难度轴：hard 题的分数上限由支点决定，写得再漂亮绕不过。
- **P2 可核验断言**：支点尽量以定量断言形式暴露（每波长采样点数、界面分辨尺度、
  守恒漂移阈值），供断言核验评分器（v0.4）机器检查。
- **P3 反模板机制**（按题选配）：欠定规格逼假设声明 / 干扰物理项逼筛选 / 算力预算逼取舍。
- **P4 私有参考与 facet cap 同步更新**，消融臂（D 系）可扩展为该题的评分器回归用例。

## 5. 与其他修订项的接口

- **→ #2（scorer v0.4）**：§2-2/2-3 是断言核验路线的动机证据；B 表"可核验断言"列
  即 v0.4 核验器的检查项清单；015/017/020 的 trap 升级属于 scorer 侧而非题面侧。
- **→ #1b（扩题）**：C 类空出 1–2 槽位；A− 若被执行数据证实（010 模式），反问题/
  数据同化类应在覆盖矩阵中扩容——现在 26 题中仅 4 道涉及反演/同化。
- **→ #3（×5 采样）**：本表全部指标重算加 CI；A− 三题的裁决依赖它与 H4 数据。
- **→ 论文（#6）**：§1 指标定义 + §2 全局诊断直接构成 item-analysis 小节；
  "difficulty 轴失效→支点原则→重写→区分度恢复"是 v0.4 的核心改进叙事。

## 6. 预期效果的可证伪判据（预注册风格）

重写完成后用同一 6 模型面板重跑：
1. 全局 facet 满分率 38.1% → **<15%**（修订计划既定目标）；
2. B/C 类题 spread 中位数 2 → **≥4**；
3. 难度层均分出现单调梯度（easy > hard_plus 至少 2 分）；
4. 重写题的任务特异 cap 在 ≥1 个模型输出上触发（trap 不再是死条款）。
未达标的题如实报告并降级为"保留-低权重"或砍除。
