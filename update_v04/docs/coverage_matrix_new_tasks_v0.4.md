## 2. 新题清单（12 题 + 2 备选,每题带出生四件套）

出生标准(与重写原则一致):P1 支点 + 任务特异 cap + 私有参考方案 +
可执行性标记(◆ = 可进未来执行效度轮次,1D/小 2D 且参考解便宜)。

### 补 C 轴（间断/自由边界）

**N1 `hard_stefan_moving_boundary_027`** ◆
一维两相 Stefan 融化问题:界面位置未知、随时间移动,界面上有 Stefan 条件
(潜热 × 界面速度 = 通量跳跃)。支点:界面的表示决策(front-tracking 的
界面参数化网络 vs 相场/水平集光滑化,各自的守恒代价);Stefan 条件作为
显式约束不可省略。cap:无界面表示机制/丢 Stefan 条件 → PC/MC 2。
参考:一维 Neumann 相似解(解析)。

### 补 D 轴（反演扩容——010 负区分度若被执行数据证实为"测到独立维度",本轴是最高优先扩容方向）

**N2 `medium_heat_robin_bc_inference_028`** ◆
从内部温度观测反演 Robin 边界系数(对流换热系数 h)。支点:边界参数
可辨识性依赖边界附近的观测布局——远离边界的观测对 h 几乎无信息;计划
必须论证观测-参数信息通路。cap:声称任意布局皆可辨识 → PF 2。
参考:1D 解析级数解。

**N3 `hard_advdiff_velocity_inverse_029`** ◆
从示踪剂浓度场反演空间变化的流速场 v(x)。支点:欠定性的结构(v 只在
∇c≠0 处可辨识——平坦区零信息),正则化选择决定恢复什么。cap:无
零梯度盲区讨论 → PF/PC 3。参考:制造解。

**N4 `hard_heat_time_varying_alpha_030`** ◆
时变扩散系数 α(t) 反演。支点:时间局部可辨识性(每个时刻的 α 只由该
时刻附近数据约束)与参数化选择(逐点 vs 基函数 vs 网络)的偏差-方差
取舍。cap:常数 α 处理 → wrong-task-type 族;无时间正则化讨论 → TS 3。

### 补 E 轴（噪声模型一等化）

**N5 `medium_noise_model_selection_031`** ◆
同一物理(1D 扩散),数据含离群点 + 异方差噪声(题面声明生成机制)。
支点:损失函数选择必须由噪声模型推出(L2 对高斯、L1/Huber 对离群、
加权对异方差),错误配对可判。cap:噪声声明与损失形式矛盾 → TS 2。
与 013 的区别:013 考噪声进目标的意识,本题考**选择的正确性**(给了
三选一的明确决策场景)。

### 补 F 轴（结构保持扩容）

**N6 `hard_schrodinger_conservation_032`** ◆
一维非线性薛定谔:质量与能量双守恒 + 复值场。支点:守恒结构的处理
(投影/惩罚/构造)+ 复表示(与 020 互补:时域复场);长时间漂移是
可判验证承诺。cap:实值化无理由 → MC 2;无守恒监控 → VR 2。
参考:孤子解析解。

**N7 `hard_plus_shallow_water_well_balanced_033`** ◆
带底地形的浅水方程,静水平衡态(lake at rest)。支点:well-balanced
性质——朴素残差在不平坦底上不保持静水解,通量与源项必须配平;计划
须承诺平衡态测试(初始化静水,看是否产生伪流动)。cap:无平衡态
讨论 → PC/VR 2。参考:平衡态解析 + 小扰动波。执行效度轮次的优质
候选(伪流动幅度是清晰的执行指标)。

### 补 H 轴（算子/代理决策扩容）

**N8 `hard_plus_parametric_family_surrogate_034`**
参数化 PDE 族(如变频源 Poisson)+ 明确的查询预算场景(将被查询
10⁴ 次不同参数)。支点:单实例 PINN vs 参数化网络 vs 算子学习的
成本-精度论证(与 026 互补:026 考"识别场景",本题给定场景考
**方案设计与成本核算**——直接服务论文的"评测成本表"叙事)。
cap:无查询成本核算 → MC/TS 3。

### 补 I 轴（适定性/约束冲突）

**N9 `medium_overdetermined_bc_conflict_035`** ◆
椭圆问题同时给了 Dirichlet 与 Neumann 全边界数据,且题面声明两者来自
不同测量、可能不相容。支点:过定问题的处理决策(最小二乘意义的解/
柯西问题正则化/一致性检验后择一),静默同时硬施加两者 = 不适定。
cap:双硬约束无不相容讨论 → PF/PC 2。这是"欠定逼假设声明"的镜像
(过定逼取舍声明),补全适定性轴。

### 补 J 轴（元方法论——PdEo"评分器测不到真理解"批评的正面回应）

**N10 `medium_inverse_crime_036`** ◆
反问题验证设计:题面给出"用同一求解器生成合成数据并验证"的诱饵场景。
支点:识别 inverse crime(验证数据与求解模型同源 → 虚假高精度),
承诺独立参考(不同离散/噪声注入/真实数据保留)。cap:接受同源验证
→ VR 2。零执行成本(纯方法论),但支点二元清晰。

**N11 `easy_calibration_control_037`** ◆
**阴性对照题**:诚实的、无陷阱的 1D 扩散(规格齐全、无缺失信息、无
干扰项)。心理测量学功能:锚定量表下限行为——在此题上全模型应得高分
且 spread 小是**预期结果**,用于校准其他题的区分度不是评分器噪声。
不设新 cap;文档标注其对照身份(入 #6 设计决策表)。

### 补 B 轴（预算约束原生题）

**N12 `hard_budget_triage_3d_038`**
3D Poisson,严格预算(显存/配点/步数题面给定,量级刻意紧张)。支点:
可行性分诊——计划必须先算"预算够不够"(配点内存 × 自动微分开销),
再给降级路线(降维切片/对称利用/低秩表示)。诚实的"不可行 + 降级
方案"得分高于虚假的"可行"。cap:无预算核算 → TS 2;声称轻松可行
且算术错误 → 断言矛盾。服务"省算力"主线卖点。
